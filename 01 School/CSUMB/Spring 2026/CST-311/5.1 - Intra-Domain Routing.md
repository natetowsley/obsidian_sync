# Network Layer: Data Plane, Control Plane
- **Data plane**:
	- Local, per-router function
	- Determines how datagram arriving on router input port is forwarded to router output port
- **Control plane**:
	- Network-wide logic
	- Determines how datagram is routed among routers along end-end path from source host to destination host
		- Routing algorithms
		- Software-defined network (SDN)

# Functions
- Two approaches to structuring network control plane:
	- **Per-router control** (traditional)
	- **Logically centralized control** (software defined networking)
## Per-Router Control Plane
- Individual routing algorithm components **in each and every** router interact in the control plane

# Internet Routing
- Internet organized as two level hierarchy
- **First level** - autonomous systems (AS's)
	- AS - region of network under a single administrative domain
	- Examples: Comcast, AT&T, Verizon, Sprint, etc
- AS's use **intra-domain** routing protocols internally
	- Distance Vector, e.g., Routing Information Protocol (RIP)
	- Link State, e.g., Open Shortest Path First (OSPF)
- Connections between AS's use **inter-domain** routing protocols
	- Border Gateway Routing (BGP)
	- De facto standard today, BGP-4
## Why Do We Need AS's?
- Routing algorithms are not efficient enough to execute on the entire internet topology
- Different organizations may use different routing policies
- Allows organizations to hide their internal network structure
- Allows organizations to choose how to route across each other (BGP)

# Routing Protocol
- **Routing protocol goal:** determine "good" paths (equivalently, routes), from sending hosts to receiving host, through network of routers
	- **Path**: Sequence of routers packets traverse from given initial source to final destination host
	- **Good**: least "cost", "fastest", "least congested"
	- A "top-10" networking challenge
- Assume:
	- A network with N nodes
	- Each node only knows
		- Its immediate neighbors
		- The cost to reach each neighbor
- How does each node learn the **shortest path** to every other node?

## Intra-Domain Routing Algorithms
- **Distance Vector**
	- Routing information protocol (RIP), based on Bellman-Ford
	- Routers periodically exchange reachability information with neighbors
- **Link State**
	- Open Shortest Path First (OSPF), based on Dijkstra
	- Each network periodically floods immediate reachability information to all other routers
	- Per router local computation to determine full routes
### Distance Vector - Bellman-Ford Algorithm
#### General Idea
- What is a distance vector?
	- Current best known cost to reach a destination from a specific node
- Example

## Link State Broadcast
![[Pasted image 20260420145010.png|450]]
- **Global (centralized) approach**: Network topology, link costs known to *all* nodes
	- Each node knows its connectivity and cost to direct neighbors
	- Each node tells every other node this information
		- This is called **Link State Information**
	- Each node learns complete network topology
- Use Dijkstra to compute shortest paths
- A node using the **link state** route construction algorithms must send link cost information to every node in the network **WITHOUT using routing tables!!!**
- **A: Flooding**
	- A basic algorithm to perform broadcasting without the use of routing tables
### Flooding Details
- Each node periodically generates **Link State Packet with the following fields:**
	- ID of node generating the LSP
	- List of direct neighbors and costs
	- Sequence number (64-bit, assumed to never wrap)
	- Time to live
- Flood is reliable (ack + retransmission)
- Sequence number "versions" each LSP
- Receivers flood LSPs to their own neighbors
	- Except whoever originated the LSP
- LSPs also generated when link state changes
#### Flooding Algorithms
1. A node select a Time To Live (TTL) value that is **large enough** to allow the packet to reach every node in the network (set to the maximum distance between 2 nodes in network)
2. A node send the message to **all** its neighbors
3. When a node send he FLOOD message, it decrements the value in the Time To Live  (TTL) field
4. If TTL > 0, the node forwards the FLOOD message to all its neighbors except the neighbor from it the message was received. If TTL = 0, the FLOOD message is **discarded**
- EX:
![[Pasted image 20260420145835.png|350]]
#### Reliable Flooding
- Nodes will only forward messages with a sequence number **higher** than it has seen before
	- Sequence number is used as version
- Nodes maintain a last version number forwarded
- The reason it is called reliable flooding is because the use of the sequence number allow nodes to identify the most recent copy of the information and prevent them from (re)transmitting old information
