- Protocol

# Network-Layer: Services and Protocols
- **Sender**: Encapsulates segments into datagrams, passes to link layer
- **Receiver**: Delivers segments to transport layer protocol
- Network layer protocols in every internet device: hosts, routers

# Network-Layer: Routers
- **Routers:**
	- Examines header fields in all IP datagrams passing through it
	- Moves datagrams from input ports to output ports to transfer datagrams along end-end path

# Two Key Network-Layer Functions
- **Forwarding:** Move packets from a router's input link to appropriate router output link
- **Routing:** Determine route taken by packets from source to destination
	- Routing algorithms
- Analogy: Taking a trip
	- Forwarding: Process of getting through single interchange
	- Routing: Process of planning entire trip

# Data Plane, Control Plane
- **Data Plane**:
	- Local, per-router function
	- Determines how datagrams arriving on router input port is forwarded to router output port
- **Control Plane**:
	- Network-wide logic
	- Determines how datagram is routed among routers along end-end path from source host to destination host
		- Routing algorithms
		- Software-defined network (SDN)
## Per-Router Control Plane

# "Best Effort" Service Model
- Simplicity and efficiency
	- A best effort approach minimizes the complexity of the protocol, making it suitable for a wide range of devices and networks
	- Connectionless
	- Lowest common denominator - ensuring interoperability in a heterogeneous network environment
- Scalability:
	- Providing strong quality of service (reliability) for every packet would require a significant amount of overhead and infrastructure, making the network less scalable and more complex
	- Instead of relying on the network to provide guarantees, higher-layer protocols (e.g., TCP) are responsible for reliability
