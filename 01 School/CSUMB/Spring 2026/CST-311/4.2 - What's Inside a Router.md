# Revisit: Switch and Router
- Mailing network analogy
	- Routers and switches: Carrier facilities including local post offices
![[Pasted image 20260323143219.png|350]]
![[Pasted image 20260323143242.png|300]]
- Main purpose: Allow the transmission of packets of data or information between intended nodes identified by IP addresses
-  Switches connect all the devices, including computers, printers, and servers, in a small business network
- Routers can connect multiple switches, and their respective networks, to form a larger network
![[Pasted image 20260323143415.png]]
- Switches work excellently for networks where multiple devices need a connection within a limited range
- Internet routers work well for homes or offices to connect to the internet where many devices may require a connection to a WAN
- In consumer products, switch and router can be combined into one device (with the modem), so a router might have a switch built in, but a router IS NOT a switch

# Input Ports
![[Pasted image 20260323143801.png|350]]
- Decentralized switching:
	- Using header field values, lookup output port using forwarding table in input port memory ("match plus action")
	- Goal: Complete input port processing at "line speed"
	- Input port queuing: If datagrams arrive faster than forwarding rate into switch fabric
		- Introduces queuing delay
- Forwarding
	- Using header field values, lookup output port using forwarding table in input port memory ("match plus action")
	- Destination-based forwarding: forward based only on destination IP address (Traditional)
	- Generalized forwarding: Forward based on any set of header field values
## Forwarding Table
- A table data structure used by a router to make decisions about where to forward incoming packets
- Includes entries that map destination IP addresses directly to the output interface or next-hop router where packets should be forwarded
- Designed for fast lookups, allowing routers to quickly determine how to forward packets as they arrive
## Routing Table
- Routing Table:
	- Store information about the network topology
	- Contains a list of available routes or paths to different destinations in the network, including the information in the forwarding table
- The forwarding table is derived from the routing table but is typically a simplified, optimized, and more efficient version
	- It contains only the information needed for quick packet forwarding

## Input Port Queueing
- If switch fabric slower than input ports combined, **queueing** may occur at input queues
	- Queueing delay and loss due to input buffer overflow
- **Head-of-the-Line (HOL) blocking**: Queued datagram at front of queue prevents others in queue from moving forward
![[Pasted image 20260406141721.png|550]]
## Output Ports
![[Pasted image 20260406141743.png|450]]
- **Buffering** required when datagrams arrive from fabric faster than link transmission rate
	- **Drop policy**: Which datagrams to drop if no free buffers?
- **Scheduling discipline** chooses among queued datagrams for transmission
	- Priority scheduling - who gets best performance
	- Network neutrality
# How Much Buffering?
- Excessive buffering in a router is not ideal for several reasons, and it can lead to various network performance issues
	- Causing increased latency (delay) in the network. Negatively impact real time applications such as voice and video calls, online gaming, and interactive web applications
	- Make it challenging for network protocols and algorithms to respond to changing network conditions. It can hinder the ability to adapt to variations in traffic loads and congestion
- RFC 3439 rule of thumb: average buffering equal to "typical" RTT (say 250 msec) times link capacity C
	- EX: C = 10 Gbps link: 2.5 Gbit buffer
## Buffer Management
- Dropping Policy: Which packet to add, drop when buffers are full
	- **Tail Drop**: Drop arriving packet
	- **Priority**: Drop/Remove on priority basis
## Packet Scheduling
- First come, first served (FCFS)
- Priority
- Round Robin (RR)
- Weighted fair queueing
### FCFS
- Packets transmitted in order of arrival to output port
	- Also Known As: First-in-first-out (FIFO) - queue data structure
### Priority
- Arriving traffic classified, queued by class
	- Any header fields can be used for classification
- Send packet from **highest** priority queue that has buffered packets
	- FCFS within the same priority class
- Priority queue data structure
### Priority Class
- ISPs may decide on the priority classes
- EX:
	- **Classification by traffic types**: Certain applications' traffics might have higher priority than other. EX: real-time application traffic could have higher priority than emails
	- Could be determined based on information of higher layer (port number)
	- **Classification based on source/destination addresses**
	- **Companies could pay the ISPs for better service**
		- Network neutrality
### Round Robin (RR)
- Arriving traffic classified, queued by **class**
	- Any header fields can be used for classification
- Cyclically, repeatedly scan class queues, sending one complete packet from each class (if available) in turn
![[Pasted image 20260406143028.png|500]]
### Weighted Fair Queueing
- Generalized Round Robin
- Each class, i, has weight, $w_i$ , and gets weighted amount of service in each cycle
![[Pasted image 20260406143136.png|400]]
### Sidebar: Network Neutrality
- Often referred to as "net neutrality"
- The principle that all internet traffic should be treated equally by internet service providers (ISPs) and governments, without discrimination or preference given to specific types of content, applications, websites, or services
- **Packet scheduling, buffer management are the mechanisms**
- Supporter arguments:
	- Prevent ISPs from filtering Internet content
	- Foster freedom of speech
	- Encouraging innovation, competition
- Opponent arguments:
	- Reduce investment
	- Prevent Internet traffic from being allocated to the most needed users
	- Could slow down the deployment of advanced networking technologies
