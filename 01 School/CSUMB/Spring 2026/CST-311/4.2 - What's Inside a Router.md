# Revisit: Switch and Router
- Mailing network analogy
	- Routers and switches: Carrier facilities including local post offices
![[Pasted image 20260323143219.png|350]]
![[Pasted image 20260323143242.png|300]]
- Main purpose: Allow the transmission of packets of data or information between intended nodes identified by IP addresses
-  Switches connect all the devices, including computers, printers, and servers, in a small business network
- Routers can connect multiple switches, and their respective networks, to form a larger network
![[Pasted image 20260323143415.png]]
- Switches work excellently for networks where multiple devices need a connection within a limited range
- Internet routers work well for homes or offices to connect to the internet where many devices may require a connection to a WAN
- In consumer products, switch and router can be combined into one device (with the modem), so a router might have a switch built in, but a router IS NOT a switch

# Input Ports
![[Pasted image 20260323143801.png|350]]
- Decentralized switching:
	- Using header field values, lookup output port using forwarding table in input port memory ("match plus action")
	- Goal: Complete input port processing at "line speed"
	- Input port queuing: If datagrams arrive faster than forwarding rate into switch fabric
		- Introduces queuing delay
- Forwarding
	- Using header field values, lookup output port using forwarding table in input port memory ("match plus action")
	- Destination-based forwarding: forward based only on destination IP address (Traditional)
	- Generalized forwarding: Forward based on any set of header field values
## Forwarding Table
- A table data structure used by a router to make decisions about where to forward incoming packets
- Includes entries that map destination IP addresses directly to the output interface or next-hop router where packets should be forwarded
- Designed for fast lookups, allowing routers to quickly determine how to forward packets as they arrive
## Routing Table
- Routing Table:
	- Store information about the network topology
	- Contains a list of available routes or paths to different destinations in the network, including the information in the forwarding table
- The forwarding table is derived from the routing table but is typically a simplified, optimized, and more efficient version
	- It contains only the information needed for quick packet forwarding
